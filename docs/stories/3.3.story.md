## Contexto

Com 1.179 arquivos `.md` no repositório, a documentação já existe — mas está dissociada do código. Sem JSDoc, a API pública não tem single source of truth: o código muda, a doc manual diverge, e ninguém percebe até alguém encontrar o bug.

Esta story foca nos arquivos de maior impacto (top 20 por uso) e configura a geração automática para que o problema não se repita.

**Depende de:** EPIC-001 — com `checkJs: true` ativo, o JSDoc se beneficia do type checking e os erros de tipo nos JSDoc são capturados em CI.

---

## Acceptance Criteria

- [ ] **AC1:** JSDoc completo nos top 20 arquivos core por frequência de uso (métodos públicos, parâmetros, tipos de retorno, @throws)
- [ ] **AC2:** Script de geração automática de API docs configurado (`npm run docs:generate`)
- [ ] **AC3:** Docs geradas em `docs/api/` — um arquivo por módulo
- [ ] **AC4:** CI/CD executa `npm run docs:generate` e falha se JSDoc está ausente em arquivos públicos
- [ ] **AC5:** README principal linkado para `docs/api/`
