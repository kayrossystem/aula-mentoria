## Contexto

O `tsconfig.json` do AIOS Core tem `checkJs: false`, desabilitando toda verificação de tipos em 243 arquivos `.js` do core. Isso torna erros de tipo (null reference, propriedade inexistente, contrato de função quebrado) completamente invisíveis até runtime.

Em um framework de orquestração de IA, erros silenciosos de tipo resultam em comportamentos inesperados de agentes em produção — difíceis de rastrear e reproduzir.

**Esta é a story com maior ROI do projeto inteiro.** Custo: 3-5 dias. Benefício: elimina uma categoria inteira de bugs em produção.

---

## Acceptance Criteria

- [ ] **AC1:** `tsconfig.json` tem `checkJs: true` (ou equivalente em `jsconfig.json` nos packages)
- [ ] **AC2:** `npm run typecheck` (ou `tsc --noEmit`) executa sem erros do tipo `TS2xxx` bloqueantes
- [ ] **AC3:** Erros de tipo não-bloqueantes (warnings) estão documentados em `docs/architecture/type-debt.md` com issue aberta para cada grupo
- [ ] **AC4:** CI passa com `typecheck` como step obrigatório
- [ ] **AC5:** Nenhum `// @ts-ignore` adicionado como atalho — toda supressão deve ser `// @ts-expect-error` com comentário justificando

## Notas para @dev (Dex)

**Estratégia recomendada (ordem de execução):**

1. Ative `checkJs: true` e rode o compilador para ver o volume total de erros.
2. Se o volume for muito alto (>100 erros), use a estratégia de `// @ts-expect-error` temporário + issue, mas documente CADA supressão. Não silencie em massa.
3. Priorize erros em arquivos com maior tráfego: `master-orchestrator.js`, `workflow-executor.js`, `bin/aiox.js`.
4. Erros em arquivos de teste podem ser tratados com menor prioridade — foco nos arquivos core primeiro.
5. **Não refatore lógica** nesta story — apenas corrija tipos. Refatoração vem na EPIC-002.

**Comandos úteis:**
```bash
# Ver todos os erros com contagem por arquivo
npx tsc --noEmit 2>&1 | grep "error TS" | sed 's/(.*//' | sort | uniq -c | sort -rn

# Ver erros apenas em arquivos críticos
npx tsc --noEmit 2>&1 | grep "master-orchestrator\|workflow-executor\|bin/aiox"
```
