## Contexto

O CLI `aiox-minimal` está marcado como deprecated mas continua operacional e acessível. Usuários podem estar usando este caminho sem saber que estão em código não mantido.

Isso cria confusão sobre qual CLI é o oficial, consome tempo de suporte (bugs reportados no CLI errado) e representa superfície de ataque adicional. A correção é simples e tem impacto direto na clareza do produto.

---

## Acceptance Criteria

- [ ] **AC1:** Ao executar `aiox-minimal`, o usuário recebe uma mensagem de aviso clara: qual é o CLI correto, data de remoção e link de documentação
- [ ] **AC2:** Após exibir o aviso, `aiox-minimal` executa o comando equivalente no CLI principal (comportamento de ponte), ou encerra com exit code 0 após aviso
- [ ] **AC3:** `package.json` tem `aiox-minimal` marcado como deprecated na seção `bin` (com comentário) e data de remoção definida
- [ ] **AC4:** README principal tem seção "CLI Deprecated" com data de remoção
- [ ] **AC5:** `CHANGELOG.md` registra a mudança
