## Contexto

O hook system está implementado 5 vezes, uma por IDE suportada (Claude Code, Codex, Gemini, Cursor, AntiGravity), sem abstração comum. Cada mudança no comportamento de hooks precisa ser replicada em 5 lugares. A inconsistência entre implementações é uma fonte contínua de bugs difíceis de isolar.

Esta story não escreve código — projeta a interface que permitirá unificar as 5 implementações via adapter pattern.

## Notas para @architect (Aria)

**Critério de boa interface:** Um dev que nunca viu o código das 5 implementações deve conseguir criar um 6º adapter apenas lendo `hook-interface.md`.

**Padrão de referência:** Adapter Pattern (GoF). Cada IDE terá seu `[ide]-hook-adapter.js` que implementa `IHookAdapter`. A lógica compartilhada vai em `base-hook-adapter.js`.

Não projete para perfeição — projete para resolver o problema das 5 implementações desconectadas agora.
