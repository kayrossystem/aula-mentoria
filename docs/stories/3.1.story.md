## Contexto

Usuários que estão em v4.31 não têm caminho documentado para upgrade. Isso fragmenta a base instalada entre versões e multiplica o custo de suporte (bugs reportados em versões antigas, features solicitadas que já existem na v5).

O `CHANGELOG.md` contém o histórico de mudanças, mas não está estruturado como guia de migração passo a passo. Esta story transforma esse histórico em um documento de upgrade acionável.

---

## Acceptance Criteria

- [ ] **AC1:** `docs/migration/v4-to-v5.md` publicado e linkado no README principal
- [ ] **AC2:** O guia cobre: breaking changes, mudanças de configuração, mudanças de CLI, mudanças de API pública
- [ ] **AC3:** Para cada breaking change: exemplo de código antigo (v4.31) e equivalente novo (v5.0.3)
- [ ] **AC4:** Seção de "Quick Start Migration" com os 5 passos mais comuns
- [ ] **AC5:** Guia revisado por @dev para precisão técnica
